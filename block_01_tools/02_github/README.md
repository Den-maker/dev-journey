# Урок 02: Git и GitHub

**Статус:** ✅ Завершён

## Что изучили

- Настройка Git (user.name, user.email)
- Инициализация локального репозитория (git init)
- Создание репозитория на GitHub
- Связывание локального и удалённого репозитория
- Основной цикл работы: add → commit → push
- Работа с ветками (master vs main)
- Файл .gitignore для Go проекта
- Решение проблемы с пустыми папками через .gitkeep

## Ключевые команды
```bash
git init                              # инициализация
git remote add origin <url>           # связка с GitHub
git add .                             # добавить всё
git commit -m "message"               # зафиксировать
git push                              # отправить на GitHub
```

## Важные моменты

- Git не отслеживает пустые папки — используем `.gitkeep`
- GitHub ожидает ветку `main`, а не `master`
- Коммиты пишем в настоящем времени: `Add`, `Fix`, `Update`
- `.gitignore` создаём до начала работы с кодом
